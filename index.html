01 - Project Motivation & Introduction

Making the web more accessible with computer vision

A vast majority of images that can be found on the web contain no alt-text: the short written descriptions that screen readers use to convey image content to visually impaired users. The goal of this project is to explore whether a modern vision-language model can generate useful alt-text automatically, and where it falls short.

What is alt-text and why does it matter?

Alt-text is an HTML attribute that allows screen readers describe images to users who can't see them. Below is an example of this tag

Without it, images are completely silent: users hear nothing, or just a raw filename.

According to WebAIM's annual accessibility survey, missing or inadequate alt-text is one of the most common accessibility failures on the web. The scale of the problem is enormous and mostly unaddressed.

WCAG 2.1 requires that all non-decorative images have a text alternative that serves the same purpose as the image. That standard is widely unmet.

What does good alt-text look like?

WCAG 2.1 (the Web Content Accessibility Guidelines) defines several principles for good alt-text:

  • It should describe the content and function of the image, not just what's visible
  • Decorative images should have empty alt attributes (`alt=""`)
  • Text in images must be included in the description
  • Context matters: the same image might need different alt-text in different settings

Based from our list above the last point is where automated tools struggle most.

Project Structure

This project asks: can a modern vision-language model generate alt-text that's actually useful? And where does it fail, and for whom? Based on these goals this site will walk through the full project in four sections

02 - How the Model Works

Understanding the Basis of how our Model

This project utilizes BLIP (Bootstrapping Language-Image Pre-training), this is a vision-language model developed by Salesforce Research. Here's how it works at a conceptual level.

From Image to Caption

BLIP has two main components that are working togther

1. A vision encoder: it takes in the raw image and converts it into a set of feature vectors that represent what's in the image. BLIP utilizes a Vision Transformer (known as ViT) for this, which splits the image into patches and processes them similarly to how a text transformer processes words

2. A language decoder: it takes those image features and generates and generates a caption word by word, conditioned on what came before. It is mainly a language model that is trained to describe images instead of predicting words in a text.

Those two components were pre-trained together on hundereds of millions of image-caption pairs from the internet.

Loading in BLIP

model_id = "Salesforce/blip-image-captioning-base" processor = BlipProcessor.from_pretrained(model_id) model = BlipForConditionalGeneration.from_pretrained(model_id) print(f"Parameters: {sum(p.numel() for p in model.parameters()):,}")

Why This Model for This Task?

The reason behind choosing the BLIP model compared to other approaches (like image classifiers) is that since alt-text requires description, not just simple labeling. Knowing that an image contains a "dog" is less useful than knowing it shows "a dog catching a frisbee at a dog park." BLIP's captioning capability is much more closer to what good alt-text requires.

Additionally, it is also freely available via Hugging Face, runs on a CPU (slowly) or on GPU (faster), and it has been well studied; signaling that there is existing literature that exists to draw on when analyzing the failure of the modes that it contains
